--- ARCHITECTURE & LOGIC CHANGES ---
1. MODULAR BACKEND ARCHITECTURE (SOLID-compliant):
   - Models layer: SQLAlchemy 2.0 ORM with async support, declarative mapped columns
   - Schemas layer: Pydantic v2 with input/output separation (prevents data leakage)
   - Routers layer: Thin controllers that delegate to services
   - Services layer: Business logic encapsulation (auth implemented, others stubbed)
   - Agents layer: AI agent classes with defined interfaces for Phase 2 implementation
   - Utils layer: Shared security + exception utilities

2. DATABASE SCHEMA (PostgreSQL):
   - 6 tables: users, job_preferences, parsed_resumes, scraped_jobs, user_jobs, benchmark_scores
   - JSONB columns for semi-structured AI outputs (parsed_data, optimized_cv, interview_prep)
   - Cross-platform dedup via description_hash (SHA-256 of first 200 chars)
   - GDPR compliance: benchmark_opt_in flag, seniority/niche snapshots, 30-peer minimum threshold
   - Check constraints on enums (seniority_level, location_type, status)
   - Composite unique constraint for dedup (company_name + job_title + description_hash)

3. FRONTEND ARCHITECTURE:
   - Vite + React 19 with React Router for SPA routing
   - Design system with CSS custom properties (dark theme, Inter font, animations)
   - API client with JWT auto-attachment via localStorage
   - Component structure: Layout shell → Pages → Hooks → Utils

4. CI/CD PIPELINE:
   - Backend: Flake8 lint → Black format check → isort import order → pytest
   - Frontend: ESLint → Prettier → Vite production build
   - Docker: Build verification for both images (runs after lint/test pass)

5. TECH STACK DECISIONS:
   - LLM: OpenAI GPT-4 (structured output, mature API)
   - Job API: Adzuna (free tier, legal, JSON responses)
   - Storage: Local filesystem with S3-ready abstraction
   - Deployment: Docker Compose (local), Railway-ready
