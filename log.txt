1. DATABASE SCHEMA IMPLEMENTATION:
   - Created 6 tables via 4 Alembic migrations (users, job_preferences, parsed_resumes, scraped_jobs, user_jobs, benchmark_scores)
   - All tables use UUID primary keys with auto-generation
   - Timestamps use timezone-aware DateTime (UTC)
   - JSONB columns with GIN indexes for efficient JSON querying (parsed_data, optimized_cv, interview_prep, missing_skills)

2. CONSTRAINT ENFORCEMENT:
   - CHECK constraints for enum validation (seniority_level, location_type, status)
   - CHECK constraints for range validation (match_score 0-100, benchmark score 0-100)
   - UNIQUE constraints for data integrity (email, external_url, user-job pairs)
   - Composite UNIQUE constraint for cross-platform job deduplication (company+title+description_hash)

3. CASCADE DELETE STRATEGY:
   - User deletion cascades to: job_preferences, parsed_resumes, user_jobs, benchmark_scores
   - Job deletion cascades to: user_jobs
   - Job deletion sets NULL on: benchmark_scores (preserves historical benchmark data)
   - All foreign keys use CASCADE or SET NULL for referential integrity

4. INDEXING STRATEGY:
   - B-tree indexes on: email (fast login), user_id (foreign keys), description_hash (deduplication)
   - GIN index on: parsed_data JSONB (efficient skill/experience queries)
   - Composite indexes on: (user_id, status) for dashboard queries, (user_id, job_id) for benchmark lookups

5. GDPR COMPLIANCE:
   - benchmark_opt_in flag defaults to false (explicit opt-in required)
   - Seniority/niche fields snapshot at calculation time for historical accuracy
   - Table comment documents GDPR measures (30-peer minimum, anonymized aggregates, no PII)

6. TEST INFRASTRUCTURE:
   - Test database (autoapply_test_db) isolated from development database (autoapply_db)
   - Session-scoped test_engine fixture creates/drops test DB
   - Function-scoped test_session fixture with automatic transaction rollback (no cleanup overhead)
