--- ARCHITECTURE & LOGIC CHANGES ---
1. JWT AUTHENTICATION DEPENDENCY:
   - get_current_user extracts Bearer token from Authorization header
   - Decodes JWT using settings.secret_key and settings.algorithm (HS256)
   - Validates token expiration timestamp (UTC timezone-aware)
   - Retrieves authenticated user from database by user_id in token 'sub' claim
   - Returns 401 Unauthorized for: invalid signature, expired token, missing user, malformed token
   - get_current_active_user chains on top for future account status checks (is_active field)

2. RATE LIMITING MIDDLEWARE:
   - IP-based rate limiting using slowapi (Flask-Limiter port for FastAPI)
   - Default limit: 100 requests per minute per IP address
   - Sliding window algorithm (not fixed intervals)
   - X-Forwarded-For header support for proxied requests
   - Returns 429 Too Many Requests when limit exceeded
   - Per-route override via @limiter.limit decorator (e.g., "5/minute" for login endpoints)
   - Rate limits stored in memory (Redis support available for production)

3. SECURITY HEADERS MIDDLEWARE:
   - SecurityHeadersMiddleware injects OWASP headers into all responses
   - X-Content-Type-Options: nosniff (prevents MIME-type sniffing)
   - X-Frame-Options: DENY (prevents clickjacking via iframe embedding)
   - X-XSS-Protection: 1; mode=block (enables browser XSS filtering for legacy browsers)
   - Content-Security-Policy: default-src 'none'; frame-ancestors 'none' (API-focused CSP)
   - Referrer-Policy: strict-origin-when-cross-origin (controls referrer information leakage)
   - Server: AutoApply API (removes version disclosure)
   - HSTS header commented out (requires HTTPS in production)

4. DEPENDENCY STRUCTURE:
   - Dependencies module provides reusable FastAPI Depends() injections
   - HTTPBearer security scheme extracts token from Authorization header
   - Database session injected via get_db dependency (from database.py)
   - Returns 403 Forbidden if Authorization header missing (HTTPBearer behavior)
   - Returns 401 Unauthorized for authentication failures

5. MIDDLEWARE INTEGRATION:
   - setup_rate_limiting(app) attaches limiter to app.state and exception handler
   - setup_security_headers(app) adds SecurityHeadersMiddleware to middleware stack
   - Middleware runs for ALL routes including error responses
   - Middleware execution order: Rate limit → Security headers → Route handler
